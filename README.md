# Тестовое задание (Frontend, React)

## 📌 Описание

Данное приложение представляет собой интерфейс для работы со списком фильмов.

Реализованы три основные страницы:

* **Приветственная страница**
* **Список фильмов** — основная страница с таблицей фильмов;
* **Жанры** — страница групп, отображающая жанры фильмов в виде карточек.

Приложение получает данные с mock-сервера (`json-server`) в формате JSON и позволяет:

* просматривать список фильмов;
* выполнять поиск и сортировку;
* добавлять и удалять фильмы;
* просматривать список жанров.

---

## ⚠️ Ограничения

* Данные хранятся в mock-сервере (`json-server`) и не синхронизируются автоматически между сущностями.
* Количество фильмов в жанрах (`moviesCount`) задаётся статически и не обновляется при добавлении/удалении фильмов.
* Отсутствует полноценная серверная логика (валидация, связи между сущностями и т.д.).

---

## 🚀 Инструкция по запуску

Приложение необходимо запускать **в двух терминалах одновременно**: отдельно backend (json-server) и frontend (React).

### 1. Установка зависимостей

```bash
npm install
```

---

### 2. Запуск backend (json-server)

Открыть первый терминал:

```bash
npm run server
```

После запуска сервер будет доступен по адресу:

```
http://localhost:3001
```

---

### 3. Запуск frontend (React-приложение)

Открыть второй терминал:

```bash
npm run dev
```

После запуска приложение будет доступно по адресу:

```
http://localhost:5173
```

---

### 4. Проверка

Убедитесь, что:

* backend запущен на `3001`;
* frontend запущен на `5173`;
* данные корректно загружаются на страницах:

  * `/list` — список фильмов;
  * `/groups` — жанры.

## Выводы по проектированию UI

В ходе этого проекта я сравнивала 2 подхода к разработке: ручное проектирование и использование LLM.

### Ручное проектирование UI

Самостоятельная разработка интерфейса занимает много времени, особенно на этапе верстки. Однако данный подход обеспечивает полный контроль над архитектурой компонентов и проекта в целом.

### Использование LLM

Применение LLM позволяет в разы ускорить написание кода, особенно при реализации типовых интерфейсов и рутинных задач.
Для получения максимально качественного результата я придерживаюсь следующего плана:

1. сформировать план решения с помощью LLM;
2. скорректировать его;
3. реализовать код на основе уточнённого плана с помощью LLM.

### Общий вывод

В современных условиях LLM является важным инструментом разработки, который позволяет:

* ускорять выполнение рутинных задач;
* получать рекомендации по архитектурным решениям;
* учиться на практике
* и выполнять сложные задачи, даже не имея большого опыта в реализации подобных задач.

При этом для поддержания качества кода необходимо:

* проводить ревью сгенерированного кода;
* реализовывать сложные и критичные части системы вручную.

Таким образом, оптимальный подход — это комбинация LLM и ручной разработки.
